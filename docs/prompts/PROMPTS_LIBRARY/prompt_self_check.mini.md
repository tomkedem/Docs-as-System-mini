# חובה  קובץ זה הוא חלק משיטת Docs as System mini  
# חובה  אין לערוך קובץ זה בתוך פרויקט פעיל
# חובה  שינוי בקובץ זה יתבצע רק בעדכון רשמי של השיטה

# בדיקה עצמית לפני המשך  

מטרה  
לוודא שהעבודה שבוצעה עומדת בכל דרישות השיטה לפני המשך פעולה

הוראות  
בדוק שהשינוי תואם את הקובץ docs/project/PROJECT_SPEC.mini.md  
בדוק שאין חריגה מהקובץ docs/architecture/ARCHITECTURE_BLUEPRINT.mini.md  
בדוק שהשינוי מתאים לשלב הרלוונטי בקובץ docs/project/IMPLEMENTATION_PLAN.mini.md  
בדוק שהשינוי אינו סותר את ההיגיון בקובץ docs/project/BUSINESS_REQUIREMENTS.mini.md  
בדוק שכל הבדיקות הרלוונטיות הוגדרו ורצות בהצלחה  
בדוק שאין קבצים שעברו שינוי בלי הצדקה במשימה  
בדוק שאין שינויים לא מכוונים בקבצים נוספים  
בדוק שקובץ הלוג docs/logs/IMPLEMENTATION_LOG.mini.md מוכן לעדכון  
בדוק שההבנה שלך עדיין עקבית עם כל המסמכים שנקראו

תנאי המשך  
אין סטיות מהמסמכים  
אין בדיקות שנכשלו  
אין ספק לגבי תוצאת המשימה

פלט מצופה  
אישור פנימי מלא שהמשימה תואמת את כל המסמכים  
מוכנות לעדכון הלוג ולשלב הבא
