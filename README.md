# ‏Docs-as-System mini  
שיטה קלה, מדויקת ואחידה לפיתוח מונחה מסמכים וסוכנים

‏Docs-as-System mini היא גרסה ממוקדת של השיטה Docs-as-System.  
היא מתאימה לצוותים ומפתחים שרוצים זרימת פיתוח ברורה, אחידה ומבוססת־מסמכים  
בלי עומס ובלי תוספות מיותרות.

המערכת בנויה סביב שלושה עקרונות:
1. המסמכים הם אמת אחת.  
2. הסוכן מבצע רק מה שמתועד.  
3. כל שינוי חייב להיות עקבי, מבוקר ומגובה לוג.

השיטה מורכבת משלושה רכיבים:
- פרומפטים (PROMPTS_LIBRARY)  
- סקריפטים (automation)  
- מסמכי מדיניות, תכנון ולוגים

כל פרומפט אחראי על חלק אחד בזרימת העבודה.  
כל סקריפט אחראי על פעולה טכנית אחת (Git).  
כל מסמך מגדיר חלק אחר מהאמת ההנדסית.

## זרימת מחזור פיתוח

1. הבנת הקשר  
2. הכנת שלב  
3. ביצוע משימה  
4. בדיקה עצמית  
5. עדכון לוג  
6. הכנת קומיט  
7. הכנת בקשת משיכה  

הסוכן לעולם לא מבצע פעולות Git ישירות.  
הוא משתמש רק בסקריפטים המאושרים, ורק דרך הפרומפטים הייעודיים.

## תקינות ותמיכה

כל קובץ כולל כותרות חובה כדי למנוע שינוי אקראי.  
כל קובץ נבחן מול מסמכי הפרויקט ומדיניות הסוכן.  
כל חריגה נעצרת ומועברת להנחיה אנושית.

## התחלה

1. שכפל את הריפו  
2. ערוך רק את מסמכי הפרויקט בתיקיית docs/project  
3. אפשר לסוכן לפעול רק דרך הפרומפטים  
4. השתמש בסקריפטים הרשמיים בלבד לפעולות Git  

השיטה מאפשרת פיתוח נקי, יציב ומתועד היטב — גם לבד, גם בצוות, וגם בסביבות מבוססות AI.
